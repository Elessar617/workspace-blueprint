## Architecture diagnosis

The repo expresses a **dual-layer architecture**: a *prose layer* (markdown maps, rule files, agent specs, routing branch docs, hook docstrings) and a *code layer* (bash hooks, JS routing brain, JSON registries). The intended design is that prose informs the agent and code enforces invariants. The failure mode is that the two layers have drifted: many invariants stated in prose are not enforced in code, and several enforcement mechanisms in code are narrower than the prose claims.

Concretely:

1. **The routing-as-tool-selection layer (#6) is prose-only.** `route.mjs` selects mandatories and dispatches them as a string in `additionalContext`. No code path verifies the model invoked them. (C1, C3.)
2. **The hook gates are tool-shaped, not path-shaped.** `block-output-without-signoff.sh` and `block-cycle-overrun.sh` are gated by tool name (`Edit|Write|MultiEdit`), so any tool that can write to disk by another name (`Bash` with `>`, `tee`, `cp`, `sed -i`) bypasses them. The gates protect against the literal `Edit` tool but not against the *act of writing to a protected path*. (C2, H2.)
3. **The memory layer admits agent monologue as durable instruction.** Instincts are session-observations distilled by a plugin observer, written to disk without user consent, and re-injected into all future sessions until manually pruned. The `trigger` field that could gate relevance is dead code. (C4, H8.)
4. **The four-agent pipeline has structural cross-cycle blindness.** Adversary and reviewer never see prior cycles. The cycle cap is 5; an unaware repeat-finding from cycle 3 in cycle 4 burns the budget. (H1.)
5. **The prose layer self-disagrees on counts and references.** Three "always-loaded" map files disagree on how many hooks, rules, skills exist. Routing branch files reference agents not in the project. Filename conventions for review reports vary across four files. (H5, H6, M1â€“M3.)

The audit's primary failure mode is therefore **Tool Discipline Failure**, with significant **Memory Contamination** as the secondary mode. Wrapper Regression is not observed (the system is well-tested at the unit level). Hidden Repair Loops are minimal (one accidental case via PostToolUse portability hook, M6). Rendering/Transport corruption is bounded (the route-inject truncation, M7).
